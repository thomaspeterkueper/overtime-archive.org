## III. Klasse A — Vollautomatisch `[R]` + `[W]`

GitHub Actions, Cron-Jobs sind etabliert. Folgendes läuft ohne menschliche Beteiligung:

- OTA-INDEX täglich regenerieren
- KXF-Validierung nach jedem Commit (ARC-0007 V-1 bis V-5)
- `[AUSSTEHEND]`-Einträge aggregieren
- `ECOSYSTEM-NEWS.json` aktualisieren
- KXF-Export triggern bei KG-Änderungen `[W]`

**Governance-Regel:** Klasse-A darf niemals Inhalte ändern, Marker setzen oder kanonisieren.

---

## IV. Klasse B — Generativ-kontrolliert `[T]`

KI-generierte Entwürfe erhalten Marker `[GENERIERT — REVIEW AUSSTEHEND]` bis zur Freigabe.

Inhaltsklassen mit niedrigem Kurationsaufwand: SSF-Lernmodule L0–L1, Lernpfade, OTA-INDEX-Einträge, Erst-Entwürfe für SCI-Dokumente.

---

## V. ECOSYSTEM-NEWS Format `[W]`

```json
{
  "schema": "ECOSYSTEM-NEWS-0.1",
  "events": [
    {
      "id": "EVT-2026-07-04-001",
      "type": "module_migrated",
      "requires_action": false
    },
    {
      "id": "EVT-2026-07-04-002",
      "type": "generated_draft",
      "requires_action": true,
      "action_type": "curator_review"
    }
  ]
}
```

Event-Typen: `module_migrated`, `module_created`, `generated_draft`, `path_generated`, `validation_failed`, `document_canonized`, `prerequisite_gap`, `kxf_export_ready`
